# LaTeX原稿のgit管理規約（1指示=1commit）

## 編集を適用したら即commit
- ユーザーの1つの指示に対応する編集が承認・適用されたら、その場でgit commitする（1指示=1commit）。
- 未commitの変更を残したまま次の指示に進まない。
- 複数ファイルにまたがる編集も、指示単位でまとめて1commitにする。
- コミットメッセージは「何をどう変えたか」が履歴だけで追える具体的な内容にする。
  - 悪い例: 「update main.tex」
  - 良い例: 「§4: 引用プレースホルダを\citeに置換」「§6: 重心の説明をxu2018dwlに合わせ修正」

## pushはcommitと切り離す
- push（特にOverleaf remote）は自動では行わず、必ずユーザーの明示的な確認をとる。
- 理由: Overleaf上で共著者がコメント・添削中はpush凍結という人間の判断が必要
  （Overleafのコメントはgitで運ばれず、pushで位置ずれ・消失しうる）。

## ユーザー自身の編集との競合を避ける
- 編集を始める前にgit statusを確認する。ユーザーの未commit変更が既にある場合は、
  自分の編集と混ざらないよう分けてcommitするか、扱いをユーザーに確認する。
- 大きな書き換え（節の再構成など）はブランチを切って作業し、
  ユーザーのレビュー後にmainへmergeする方式を提案する。
