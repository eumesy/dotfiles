# 参考文献・書誌情報の規約

## 書誌情報は公式ソースのみを典拠にする

書誌情報（著者名、タイトル、誌名、巻・号・ページ、出版年、出版社）は、
以下の公式ソースだけを頼って確認・記入する。

- **使ってよいソース**
  - 出版社・学会の公式サイト（目次ページ等）
  - DOI（doi.org 経由でランディングページを確認）
  - 日本語文献: CiNii Research（cir.nii.ac.jp）、NDLサーチ
  - 情報系の国際会議・論文誌: dblp
  - arXiv 論文: arXiv の公式アブストラクトページ
- **使ってはいけないソース**
  - 手元の PDF から抽出したテキストや、PDF のファイル名からの推測
    （例: `2510-58.pdf` という名前から「58 ページ開始」と推測するのは不可）
  - 通販サイト・書評サイト等の二次情報（号数の当たりを付ける参考程度に留める）
  - モデル（Claude）の記憶だけで書誌情報を埋めること
- **理由:** PDF 抽出・ファイル名・記憶からの推測は、巻号・ページ番号・年の
  取り違えが起きやすく、しかも誤りに気づきにくい。

## 確認できなかった項目の扱い

- 公式ソースで確認できなかった項目は推測で埋めず、エントリ行末に `% 要確認` を
  付けてユーザーに確認する（この運用は既存 bibliography のヘッダ規約に合わせる）。

## 書式

- エントリの書式（区切り記号、Vol./No./pp. の表記、和文著者名の区切り等）は、
  同一文書内の既存エントリに厳密に合わせる。

## caption 等の moving argument 内の \cite

古い和文クラス（sci209 等）では `\cite` が fragile で、`\caption{...}` 内に生で書くと
`Argument of \@caption has an extra }` で落ちる。対処は次のどちらか。

- **推奨: プリアンブルで一度だけ robust 化**（書き忘れがなくなる）
  ```latex
  % \cite を robust 化（このクラスの \cite は fragile で，caption 等の
  % moving argument 内で \protect なしに使うとエラーになるため）
  \MakeRobust{\cite}
  ```
- 代替: 使用箇所ごとに `\protect\cite{...}`（グローバルな副作用ゼロだが毎回必要）
- いずれの場合もビルドして通ることを確認する。`\listoffigures` を使う文書では
  図目次の出力も確認する。

## 実例

『数理科学』掲載記事のページ範囲を手元 PDF から推測しようとして修正を受けた。
CiNii Research（crid.nii.ac.jp）で「数理科学 63(10), pp.33–41, 2025」と確定できた。
雑誌記事は出版社サイトの目次にページが載らないことが多く、CiNii が確実。
